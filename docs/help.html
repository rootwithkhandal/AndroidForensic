CLI Flag / Option Forensic Action & Evidence Artifacts Extracted
--call-logs Acquires incoming, outgoing, missed, and rejected calls (`content://call_log/calls`, `dumpsys calllog`) into SQLite/CSV/JSON (`call_logs.db`).
--messages Extracts SMS (`content://sms`) and parses WhatsApp Companion (`whatsapp_companion_raw.json`), Telegram, and Signal caches (`messages.db`).
--contacts Queries system contacts (`content://com.android.contacts/data/phones`) and merges WhatsApp JID profiles (`contacts.db`).
--connectivity Dumps Wi-Fi state (`dumpsys wifi`), Bluetooth, paired devices, and active socket connections (`connectivity_info.db`).
--notifications Captures unredacted live and historical system notification queue (`dumpsys notification --noredact`) into `notifications.db`.
--calendar Extracts user calendar events, locations, and descriptions (`content://com.android.calendar/`) into SQLite and standard `calendar_export.ics`.
--journey Harvests GPS (`dumpsys location`), EXIF photo coordinates, and builds interactive Leaflet Maps (`journey_map.html`) & Google Earth (`.kml`).
--network Extracts network interfaces (`ip addr show`), hardware MACs (`wlan0`), live sockets (`ss/netstat`), ARP neighbors, and routing tables (`network_interfaces.db`).
--browser-history Extracts search queries (`q=...`) and visited URLs across Chrome, Samsung Internet, Firefox, Edge, Brave, and Opera (`browser_visited_urls.db`).
--cookies Harvests HTTP cookies from browser profile DBs (`Cookies`, `cookies.sqlite`) and Android System WebViews into `cookies_netscape.txt`.
--device-events Reconstructs minute-by-minute timeline including Screen Lock/Unlock (`KEYGUARD`), app transitions (`usagestats`), and system reboots (`bootstat`).
--connected-devices Catalogs USB accessories (`dumpsys usb`), Bluetooth gear (`MAC`, `Name`), Wi-Fi P2P peers, Hotspot clients, companion wearables, and cast TVs.
--output / -o Specifies the custom evidence directory path (defaults to `evidence/CASE-YYYYMMDD-HHMMSS`).
--dry-run Shows default logical-acquisition paths without pulling files. Optional extractors are not run in dry-run mode.
--skip-media Skips pulling DCIM photos, videos, and large media files to accelerate rapid triage.
--skip-apk Skips extracting third-party APK packages from the device during logical sweep.

💡 Rapid Troubleshooting Helpdesk

⚠️ ADB Device Shows "unauthorized" or "offline"
When connecting the phone via USB, make sure the phone screen is unlocked. A prompt will appear stating "Allow USB debugging?". Check "Always allow from this computer" and tap **Allow**. If no prompt appears, run `adb kill-server` followed by `adb devices` to re-trigger the RSA authentication prompt.
⚠️ Access Denied when attempting `/data/data/` direct pull
By design, `/data/data/` is strictly isolated on non-rooted Android OS distributions. If you receive permission errors during raw folder copy, make sure you run our dedicated extraction flags (`--journey`, `--browser-history`, `--cookies`). Our engines automatically utilize Content Providers and system diagnostic services (`dumpsys`) to extract those artifacts without requiring root.
